# Contributing

llm-primer is early and the surface is deliberately small. If you'd like to help:

- **Issues:** open one for anything non-trivial before coding. Discussion first saves time.
- **PRs:** keep them focused. One concern per PR.
- **Style:** match the existing bash. No bashisms that break on older macOS defaults.
- **Testing:** run `primer start` and `primer` on a real Claude Code setup before submitting.

## What's open

Weekly features ship as releases. Good starter issues live at:
https://github.com/asakin/llm-primer/issues

If you want to propose something bigger (a rewrite, a new CLI backend, a different architecture), open an issue tagged `proposal` and we'll talk before you write code.
